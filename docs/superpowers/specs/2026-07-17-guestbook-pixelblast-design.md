# 留言板 PixelBlast 整页背景设计

日期：2026-07-17
状态：待用户复核
适用页面：`/guestbook`

## 1. 背景

当前留言板使用 Waline 承担登录、留言展示、发布与管理。此次改造只增强留言板的视觉体验：将 React Bits 的 `PixelBlast` WebGL 效果作为整页动态背景，同时保留现有 Waline 功能和数据路径。

用户已确认：

- 效果覆盖整个留言板页面；
- 颜色跟随网站浅色、深色模式；
- 使用完整交互，包括持续动画、鼠标流体扰动和点击涟漪；
- 采用原版 React Bits 组件方案，而不是重新仿制。

## 2. 目标

- 让 `/guestbook` 具有独立、鲜明且具有互动感的整页视觉背景。
- 保证 Waline 的登录、输入、发布、翻页和管理逻辑不受影响。
- 浅色与深色模式下均保持足够的文字对比度。
- 仅在留言板页面加载 React 和 WebGL 相关代码，不增加其他页面的客户端负担。
- 在移动设备、减少动态效果偏好和 WebGL 不可用时提供安全退化。

## 3. 非目标

- 不修改 Waline 服务端、数据库、用户体系或管理后台。
- 不改变留言板的固定路径 `/guestbook`。
- 不重新设计 Waline 的登录界面。
- 不把该动态背景扩展到博客、首页或其他页面。
- 不在本次改造中新增点赞、通知或其他业务功能。

## 4. 视觉设计

### 4.1 背景层

PixelBlast Canvas 固定覆盖浏览器可视区域，并位于留言板全部内容之后。页面滚动时背景保持在原位，形成连续的互动空间，而不是随内容重复或断开。

Canvas 使用透明渲染，下面保留一层主题渐变作为基础底色和加载、失败时的视觉兜底。

### 4.2 主题配色

- 浅色模式：低饱和青紫色粒子，背景保持明亮、轻盈。
- 深色模式：降低亮度的蓝紫色粒子，背景使用深蓝灰基底。
- 切换网站主题时，PixelBlast 颜色无需刷新页面即可同步更新。

具体色值在实现时结合现有网站主题变量进行视觉校准，不单独创建与网站割裂的新色板。

### 4.3 内容表面

留言板标题、说明和 Waline 评论区域位于半透明磨砂内容层上：

- 使用与主题相适配的半透明背景；
- 保留轻微边框和阴影，用于从动态背景中分离内容；
- 圆角、间距和字体延续网站现有设计语言；
- 不用高模糊度或过强阴影遮盖 PixelBlast 的视觉特征。

页面内容必须在动画最亮、最密集的区域仍保持清晰可读。

## 5. 交互设计

启用 PixelBlast 的完整互动能力：

- 粒子动画持续运行；
- 鼠标或触摸移动产生流体扰动；
- 点击或轻触产生扩散涟漪；
- 多次点击允许形成多个短暂涟漪。

Canvas 本身不抢占指针事件。页面级指针坐标传递给动画，因此用户操作 Waline 输入框、按钮和链接时不会被背景阻塞。背景反馈只能作为视觉响应，不得改变点击目标或表单行为。

## 6. 技术方案

### 6.1 Astro 与 React Island

项目继续以 Astro 为主，仅为留言板新增 React 支持：

- 使用 `@astrojs/react` 注册 React 集成；
- 使用 `react` 与 `react-dom` 承载 PixelBlast；
- 使用 `three` 和 `postprocessing` 提供 WebGL 渲染与后期处理；
- PixelBlast 以页面级 React Island 的形式只在 `/guestbook` 客户端加载。

由于组件依赖浏览器和 WebGL API，应采用仅客户端渲染方式，避免服务端执行访问 `window`、Canvas 或 WebGL 的代码。

### 6.2 组件边界

预计拆分为：

- PixelBlast React 组件：保留 React Bits 的着色器和动画主体；
- 留言板背景适配层：负责主题监听、页面级指针事件、减少动态效果和失败退化；
- 留言板页面样式：负责背景层级与 Waline 磨砂内容层。

原组件允许进行必要的站点适配，但不重写其核心着色器算法。

### 6.3 层级与事件

页面层级从后到前为：

1. 主题基础渐变；
2. 固定 PixelBlast Canvas；
3. 留言板内容与 Waline；
4. 网站导航、弹窗等原有高层级组件。

Canvas 设置为不接管点击，适配层监听页面级指针事件并换算为标准化坐标。事件监听在组件卸载时必须完整清理。

### 6.4 Waline 保持不变

现有调用继续使用：

- `path="/guestbook"`；
- 现有 Waline 服务地址和环境变量；
- 现有登录要求、评论展示与后台管理配置。

本次变化不得导致旧留言进入新线程，也不得创建新的 Waline 数据路径。

## 7. 性能策略

用户选择完整效果，因此桌面和移动端均保留流体与涟漪。为避免整页背景造成不必要负担：

- 限制设备像素比，避免高分辨率手机按原生像素全量渲染；
- 延续组件对 Canvas 尺寸变化的按需更新；
- 页面隐藏时暂停或停止动画帧，恢复可见后继续；
- 限制涟漪数量和生命周期；
- 只在留言板路由下载和初始化 WebGL 代码；
- 不因主题切换重复创建 Renderer 或事件监听器。

首轮参数以 React Bits 示例为基础，并在实际页面上校准像素大小、密度、速度和流体强度。视觉效果不能以明显卡顿、输入延迟或滚动抖动为代价。

## 8. 可访问性与退化

- 当系统启用 `prefers-reduced-motion: reduce` 时，不运行持续 WebGL 动画，显示静态主题渐变。
- 当 WebGL 初始化失败、浏览器不支持相关能力或组件运行异常时，页面仍显示静态主题渐变和完整 Waline 内容。
- 装饰性 Canvas 不进入键盘焦点顺序，不向屏幕阅读器重复宣读。
- Waline 的表单标签、按钮、键盘操作和焦点样式保持原状。
- 浅色和深色模式均检查正文、输入框和按钮的对比度。

## 9. 错误处理

- WebGL 初始化应被隔离，错误不得导致留言板页面整体白屏。
- 组件卸载时释放 Renderer、Composer、材质、几何体和事件监听器。
- 主题监听器、媒体查询监听器和动画帧必须在卸载时清理。
- 静态渐变默认存在，因此加载期间不会出现空白闪烁。

## 10. 验证范围

实现后至少验证：

- `/guestbook` 可以正常构建并打开；
- 浅色、深色模式的颜色切换即时生效；
- 鼠标移动、触摸移动和点击涟漪正常；
- 评论框可聚焦、可输入，登录和发布入口可点击；
- Waline 仍使用 `/guestbook` 数据路径；
- 页面滚动、导航和移动端布局正常；
- 减少动态效果模式显示静态背景；
- 模拟 WebGL 不可用时页面和 Waline 仍可使用；
- 离开留言板后，其他页面不加载或运行 PixelBlast；
- 自动化测试、类型检查和生产构建通过。

## 11. 验收标准

满足以下条件即可验收：

1. 留言板整页拥有跟随明暗主题变化的 PixelBlast 动态背景。
2. 持续动画、流体扰动和点击涟漪均可见。
3. 动态背景不拦截 Waline 或网站导航的任何操作。
4. Waline 的现有留言、登录和管理功能保持不变。
5. 内容在两种主题下均清晰可读。
6. 减少动态效果和 WebGL 失败场景具有可用的静态退化。
7. PixelBlast 依赖和运行时仅影响留言板页面。
